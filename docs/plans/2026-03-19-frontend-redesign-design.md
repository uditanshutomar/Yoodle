# Yoodle Workspace Redesign — Design Document

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:writing-plans to create the implementation plan from this design.

**Goal:** Redesign Yoodle's entire frontend from a meeting-focused app into a full workspace platform with customizable dashboard, smart Kanban board, ambient AI, and the Yoodler AI companion — all wrapped in Yoodle's signature neo-brutalist design language.

**Branch:** `feat/frontend-redesign` (original implementation preserved on `feat/product-redesign`)

---

## Decisions Log

| Question | Answer |
|----------|--------|
| Scope | Full experience redesign — new information architecture, navigation, page concepts |
| Navigation model | Hub + contextual — minimal sidebar, each top-level page is a hub with internal tabs |
| Dashboard | Customizable widget grid (react-grid-layout), smart defaults, user-persisted layout |
| AI integration | Ambient (woven everywhere) + Yoodler companion (global, context-aware) |
| Meeting room | Focused minimalism + Yoodler Live (AI is the only escape hatch) |
| Visual identity | Keep full neo-brutalist personality — thick borders, #FFE600, bold fonts, mascot |
| Mobile strategy | Mobile-first for key flows (join meeting, check tasks, view briefs), desktop-optimized for rest |
| AI companion name | **Yoodler** |

## 2. Navigation — Hub + Contextual

### Top Bar
- **Left:** Space switcher dropdown (current Space name + chevron)
- **Center:** Search bar ("Find anything")
- **Right:** Yoodler button (floating, always accessible), Pings bell, avatar menu (Preferences, profile, logout)

### Sidebar (5 items)
Collapsed to icons on medium viewports, hamburger drawer on mobile.

```
The Desk     (grid icon)
Rooms        (door icon)
The Board    (kanban icon)

Pulse        (chart icon)
```

Each hub has internal tabs for deeper navigation — no deep sidebar nesting.

### Contextual Navigation
Each hub manages its own internal navigation:
- **Rooms:** tabs for Upcoming | Past | Blueprints

- **Chatter:** tabs for Direct | Channels
- **Pulse:** sections for Trends | Vibe Checks | Heads Ups | Space Summary

## 3. The Desk — Customizable Widget Dashboard

### Layout Engine
- `react-grid-layout` with 12-column responsive grid
- Drag, drop, resize in edit mode
- Layout persisted to user preferences via `PATCH /api/users/me`
- Layout config shape:
```json
{
  "deskLayout": [
    { "widget": "up-next", "x": 0, "y": 0, "w": 8, "h": 3 },
    { "widget": "launchpad", "x": 8, "y": 0, "w": 4, "h": 2 },
    { "widget": "sticky-board", "x": 8, "y": 2, "w": 4, "h": 3 }
  ]
}
```

### Default Layout

```
+----------------------------+------------+
|         Up Next            | Launchpad  |
|    (upcoming meetings,     | (create,   |
|     quick-join, briefs)    |  join, new)|
+----------------------------+------------+
|    Yoodler Says    |   Sticky Board     |
|  (AI nudges)       | (compact tasks)    |
+--------------------+--------------------+
|   Pulse Check      |       Buzz         |
|  (mini trends)     |  (unread msgs)     |
+--------------------+--------------------+
```

### Widget Architecture
Each widget is a self-contained React component that:
1. Declares metadata: `{ id, title, icon, minW, minH, defaultW, defaultH }`
2. Receives dimensions via props and renders responsively
3. Fetches its own data on mount (lazy-load)
4. Follows Yoodle design language: thick borders, card shadows, bold headings

### Edit Mode
- "Make it yours" button in top-right of The Desk
- Activates: drag handles on widgets, resize corners, "Add Widget" drawer (slides from right)
- Widget catalog shows all available widgets with preview thumbnails
- Save/cancel buttons replace the "Make it yours" button during edit mode
- Layout auto-saves on exit

### Mobile
- Widgets stack in single column, priority-ordered (Up Next first)
- No drag/drop on mobile — fixed stack order
- Each widget renders its mobile-optimized variant

## 4. Rooms — Meetings Hub

### Internal Tabs
- **Upcoming:** Meeting cards with quick-join button, participant avatars, countdown timer, Vibe Check badge (if AI data exists), The Rundown preview chip
- **Past:** Meeting cards with Vibe Check score, To-Do count badge, click to open The Recap
- **Blueprints:** Saved meeting configurations with "Use this" button

### Room Actions (pinned at top)
- "Start a Room" — create new meeting room
- "Join a Room" — enter room code/link

### The Room (In-Meeting Experience)
- **Full-screen focused:** No sidebar, no workspace chrome, no navigation
- **Core UI:** Video/audio grid, participant list, screen share, controls bar
- **Yoodler Live:** Slide-out panel (right side) — the ONE sidebar
  - Real-time copilot suggestions stream in
  - Can fetch anything on demand: "Show my tasks," "Pull up last week's standup notes," "What did we decide about the API?"
  - Context-aware: knows current meeting, participants, agenda
  - The Yoodle mascot avatar at the top of the panel

### The Lobby (Pre-Meeting)
- Shows The Rundown automatically (AI brief)
- Participant list, meeting details, agenda
- "Enter The Room" button
- Audio/video preview and device selection

### The Recap (Post-Meeting Detail)
- Tabbed view: **Summary | To-Dos | Calls | Analytics | Transcript**
- Summary: AI-generated meeting brief
- To-Dos: Action items with "Send to Board" button (creates Stickies)
- Calls: Decisions made, with context
- Analytics: Full Vibe Check breakdown, participation metrics
- Transcript: Searchable transcript with timestamps

## 5. The Board — Smart Kanban

### Board View (default)
- **Lanes:** Configurable status columns (default: To Do | In Progress | Done)
- **Stickies:** Draggable task cards within and between lanes
- **Sticky card contents:** Title, assignee avatar, due date chip, priority badge (color-coded), source tag ("From: Monday Standup" for Auto-Stickies)

### Auto-Stickies
- Yoodler extracts action items from meetings and creates Stickies automatically
- Tagged with source meeting for traceability
- Assigned to mentioned team members when possible
- Appear in "To Do" lane with an "Auto-Sticky" badge (sparkle icon)

### List View (toggle)
- Flat table/list view for people who prefer it
- Sortable columns: title, assignee, due date, priority, status, source
- Default view on mobile

### Filters & Grouping
- Filter by: Space, assignee, due date range, priority, source meeting
- Group by: Status (default), assignee, meeting, due date

### Google Tasks Integration
- Existing Google Tasks sync bidirectionally with Stickies
- Google Tasks appear with a Google icon badge

### Mobile
- List view by default on phone
- Board view with horizontal scroll on tablet
- No drag-drop on phone — use dropdown to change lane

- **Direct:** 1-on-1 conversations
- **Channels:** Space-scoped group channels

## 7. Pulse — Analytics Hub

### Sections (scrollable page, not tabs)

**Trends**
- Meeting frequency over time (bar chart)
- Average Vibe Check scores over time (line chart)
- Duration trends
- Range selector: Week | Month | Quarter

**Vibe Checks**
- Per-meeting scores with drill-down
- Score breakdown: participation, decision quality, action item follow-through
- Color-coded: green (70+), yellow (40-69), red (<40)

**Heads Ups**
- AI-detected patterns displayed as alert cards
- Severity-coded: info (blue), warning (yellow), critical (red)
- Examples: "Mondays have 40% lower engagement," "3 meetings this week had no agenda"

**Space Summary**
- Admin/workspace-level insights
- Crew activity, meeting participation rates
- To-Do completion rates across the Space

## 8. Yoodler — AI Companion

### Global Access
- Floating button (bottom-right on desktop, bottom-right on mobile)
- The Yoodle mascot as the button icon — bouncy animation on hover
- Click to open Yoodler panel (slide-up on mobile, slide-left on desktop)

### Context Awareness
Yoodler knows where you are and adapts:
- **On The Desk:** "You have 2 meetings today with no agenda set"
- **In Rooms hub:** "Here's The Rundown for your next meeting"
- **On The Board:** "3 Stickies are overdue — want me to flag them?"
- **In The Room:** Becomes Yoodler Live — full copilot mode
- **In Pulse:** Explains trends in natural language

### Capabilities
- Fetch any workspace data on demand
- Summarize past meetings
- Surface relevant To-Dos and Calls
- Answer questions about workspace history
- Proactive nudges (Yoodler Says)

### Ambient AI Layer
Woven into every surface without requiring the companion panel:
- **Meeting cards:** Vibe Check score badge, To-Do count chip
- **Stickies:** AI-suggested priority icon (subtle sparkle)
- **The Desk:** Yoodler Says widget with proactive suggestions
- **Pulse:** Heads Up alert cards
- **The Lobby:** The Rundown auto-surfaces

## 9. Visual Identity — Neo-Brutalist Workspace

### Design Tokens (unchanged from current)
- **Primary accent:** `#FFE600` (Yoodle yellow)
- **Borders:** 2px solid, `var(--border-strong)`
- **Shadows:** Hard offset shadows `var(--shadow-card)`
- **Border radius:** `rounded-xl` (12px) for cards, `rounded-2xl` (16px) for modals
- **Heading font:** `var(--font-heading)` — bold, chunky
- **Body font:** `var(--font-body)` — clean, readable
- **Animations:** Framer Motion springs (`stiffness: 200-500, damping: 25-30`)

### Color System
- **Vibe Check green:** `#22C55E` (score >= 70)
- **Vibe Check yellow:** `#F59E0B` (score 40-69)
- **Vibe Check red:** `#EF4444` (score < 40)
- **Yoodler purple:** `#A855F7` (AI accent color)
- **Error red:** `#FF6B6B`
- **Info blue:** `#3B82F6`

### Component Patterns
- **Cards:** Thick border, hard shadow, bold header with icon
- **Buttons:** Filled yellow primary, outlined secondary, bold font
- **Badges:** Pill-shaped, color-coded, small bold text
- **Modals:** Full-screen on mobile, centered with backdrop on desktop
- **Inputs:** 2px border, rounded-xl, yellow focus ring
- **Toggle:** Yellow track when active, spring-animated thumb

### Theme Support
- **Day** (light), **Night** (dark), **Auto** (system)
- All colors via CSS custom properties
- Theme persisted to user preferences

## 10. Mobile Strategy — Key Flows First

### Mobile-Optimized Flows
1. **Join a Room** — One-tap from Up Next widget or Rooms hub
2. **The Rundown** — Full-screen AI brief before entering The Room
3. **Check Stickies** — List view of The Board, tap to update status
4. **Chatter** — Full-screen conversation threads
5. **The Desk** — Stacked widgets, read-only arrangement

### Desktop-Only Features
- Widget drag/drop customization (The Desk edit mode)
- Kanban drag-drop (The Board — board view)
- Full Pulse charts with hover interactions
- Side-by-side panels

### Responsive Breakpoints
- **Mobile:** < 640px — single column, hamburger nav, stacked everything
- **Tablet:** 640px - 1024px — sidebar icons, two-column where appropriate
- **Desktop:** > 1024px — full sidebar labels, multi-column layouts, all features
